## What This Project Is

**SkillPack** aggregates AI agent skills from skills.sh and GitHub, categorizes them by developer role, and serves them as curated bundles with a single install command per platform (Claude Code, Cursor, Copilot, Continue.dev, Universal).

- ~1,500 Tier 1 skills stored fully in PostgreSQL (quality_score ≥ threshold)
- ~110k Tier 2 skills indexed by reference, live-fetched from GitHub on demand
- 13 curated bundles (12 role-based, 1 task-based) generated by the pipeline

## Commands

### Backend

```bash
cd backend

# Install dependencies
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt

# Run API (dev)
python3 -m uvicorn api.main:app --host 0.0.0.0 --port 8000

# Run crawl pipeline
python3 run_crawl.py           # Full: skills.sh + GitHub + tagging + bundles
python3 run_crawl.py --test    # Skills.sh only (fast, no GitHub)
python3 run_crawl.py bundles   # Regenerate bundles only

# Run Celery worker (async jobs)
celery -A workers.celery_app worker --loglevel=info
```

### Frontend

```bash

```

Environment: `frontend/.env.local` — only `NEXT_PUBLIC_API_URL=http://localhost:8000` required.

## Architecture

### Data Flow

```
skills.sh scraper ─┐
                   ├─→ raw skills → tagger.py (Groq LLM) → ingestion.py → PostgreSQL
GitHub crawler ────┘                                                           │
                                                                               ↓
                                              bundle_generator.py → Bundle + BundleCommand rows
                                                                               │
                                                                               ↓
                                                                    FastAPI → Next.js frontend
```

### Tier 1 vs Tier 2

| | Tier 1 | Tier 2 |
|---|---|---|
| Storage | Full row in `skills` table | Lightweight row in `skills_index` |
| Content | `raw_content` stored in DB | Fetched live from GitHub on demand |
| Threshold | `install_count >= TIER1_MIN_INSTALLS` (default 50) | Everything else |
| API | `/api/skills/{slug}` | `/api/live/{owner}/{repo}` |

Tier 2 live-fetches are Redis-cached for 24 hours.

### Key Data Models (`db/models.py`)

**`Skill`** — The core entity (Tier 1 only):
- `slug`: `owner/repo` or `owner/repo/skillId` (3-part for skills.sh)
- `primary_category`: one of `frontend, backend, fullstack, devops, ml-ai, security, database, testing, cloud, mobile, data-science`
- `platforms`: array, default `["claude_code"]`
- `quality_score`: 0–10 float, computed by AI tagger
- `content_hash`: MD5 of `raw_content` — used to skip re-processing unchanged skills

**`Bundle`** → `BundleCommand` (one row per platform):
- `bundle.commands?.[platform]` on the frontend = `bundle_commands.command` for that platform
- Commands are generated by `pipeline/install_generator.py` as multi-slug npx commands

**`CrawlJob`** — Audit trail for every crawl execution.

### API Routes (`api/routes/`)

| Prefix | File | Notes |
|---|---|---|
| `/api/bundles` | `bundles.py` | List, get by slug, install command + increments `install_count` |
| `/api/skills` | `skills.py` | List with filters, categories, get by `/{slug:path}` |
| `/api/search` | `search.py` | Full-text (`tsvector`), fuzzy (`ILIKE`), AI bundle match (Groq) |
| `/api/live` | `live.py` | Live-fetch Tier 2 skills, Redis cache 24hr |
| `/api/crawl` | `crawl.py` | Admin-only (`X-Admin-Token` header), triggers Celery tasks |

`redirect_slashes=False` is set on the FastAPI app — routes must be called **without** trailing slash.

### Frontend Routes (`frontend/src/app/`)

| Route | Key behavior |
|---|---|
| `/` | Rotating placeholder search → `/api/search/match-bundle` → redirects to best bundle |
| `/explore` | Tabs: bundles (type filter) / skills (live search). AnimatePresence tab transitions. |
| `/bundle/[slug]` | Platform selector drives which `commands[platform]` is shown. Skills expand in-place. |
| `/skills/[...slug]` | Catch-all for `owner/repo/skillId` 3-part slugs. Quality score bar, raw SKILL.md toggle. |

`api.ts` is the single typed client. All page components are `"use client"` using `useEffect` for data fetching.

### Crawlers

**`skills_sh.py`**: Parses `__next_f` RSC JSON from skills.sh HTML to extract `initialSkills` array. Merges SSR top-600 + trending API pages (200/page) → deduplicates → top 1,500 by install count. Per-skill SKILL.md discovery via GitHub tree API with fuzzy path matching (`_find_best_skill_path`).

**`github_crawler.py`**: Uses GitHub Search API (`filename:SKILL.md`) with rate-limit aware pagination.

### Pipeline (`pipeline/`)

- `tagger.py`: Calls Groq with batches of skills, extracts JSON classification. Falls back to keyword heuristics if LLM fails.
- `bundle_generator.py`: Matches skills against predefined role/task keyword lists. Stores ordered `skill_ids` on the bundle.
- `install_generator.py`: Builds platform-specific commands (e.g., `claude skills add owner/repo skill1 skill2`).

## Important Patterns

**Bulk ingestion**: `db/ingestion.py` uses `bulk_insert_mappings` + batch `IN` queries. Do not revert to per-row `db.commit()` — it causes 300s+ timeouts on Supabase.

**3-part skill slugs**: skills.sh skills have slugs like `owner/repo/skillId`. The API route uses `/{slug:path}`, and the frontend uses `[...slug]` to reassemble with `.join("/")`.

**No trailing slashes**: `app = FastAPI(redirect_slashes=False)`. All router list endpoints use `@router.get("")` (not `"/"`).

**Content dedup**: `ingestion.py` computes MD5 hash of `raw_content` and skips re-tagging if `content_hash` matches existing row.

**Framer Motion**: All pages use `"use client"` + Framer Motion for entrance animations. Variant `transition` objects must not include `ease` as a plain string or `number[]` — use `motion.div` `transition` prop instead (Framer Motion TypeScript limitation).
